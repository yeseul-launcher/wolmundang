# 월문당 디자인 규칙

프로젝트 전반에 적용하는 디자인 원칙. 새 컴포넌트 만들 때 먼저 확인할 것.

## 카드 / 서피스

- **콘텐츠 카드에 보더 금지.** 후기 카드, 결과 카드 등 콘텐츠를 담는 카드는 테두리(border) 쓰지 말 것. 대신 반투명 채움(예: `bg-white/6`)과 섀도우로 면을 구분한다.
- 채움 우선(fill-first). 요소 구분은 보더보다 배경 채움/명도 차이로.

## 버튼

- **서브 버튼(이전 등)은 인풋과 다른 디자인.** 인풋은 보더 있는 투명 스타일, 서브 버튼은 보더 없는 반투명 채움(`bg-white/10`)으로 확실히 구분한다.
- 주 버튼(CTA/다음): `bg-primary`(#5285D4) + 흰색 + `font-semibold` + `rounded-lg`.
- 이전/다음 네비: 이전은 좁게(약 30% 너비) 왼쪽, 다음은 나머지 폭.

## 입력(폼)

- number 인풋의 위아래 스피너 버튼은 항상 숨긴다 (globals.css 전역 처리).
- 양/음력 같은 보조 선택은 생년월일 인풋 오른쪽에 좁은 세그먼트 토글로. 보더 없이 채움.

## 타이포 / 폰트

- 기본: Pretendard (`--font-sans`), 본문 SUIT (`--font-suit`).
- 디스플레이(붓글씨): 히어로 큰 타이틀은 영월체(Yeongwol), 캐릭터 대사/서브카피는 솔뫼김대건체(SolmoeKimDaeGeon).
- 탑바 텍스트는 Pretendard(붓글씨 아님).

## 컬러 토큰

- 배경 `#272727`, 히어로 배경 `#1a1a1f`, primary `#5285D4`.
- 다크 배경 기준. 텍스트 흰색 계열 + 투명도(`text-white/70` 등).

## 모션

- 로테이팅 토스트: 메시지 전환 시 slide-up + 살짝 오버슈트 팝(`toastPop`), 버블은 은은한 플로팅(`toastFloat`).
- 히어로 서브카피 둘째 줄: `fadeSlideUp` 지연 등장.
